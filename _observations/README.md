# `_observations/` — frontend continuity surface

This directory mirrors the engine repo's `_observations/` convention. It is the
frontend's running record of decisions made, observations captured, and
session briefs preserved — so context does not die in chat logs between
sessions.

## What lives here

- **Session briefs / observation memos**, named `OBSERVATIONS_YYYY_MM_DD_*.md`.
- Decision records and audit-style findings specific to the frontend repo.

## What this is not

- Not a spec, not a roadmap, not a binding doctrinal document.
- Observation-only. These memos record state; they do not commission work.
- For the ordered milestone state of the frontend, see `../ROADMAP.md`.
- For setup and operational steps, see `../SETUP.md`.

## Cross-repo note

The engine repo (`weichert/squadvault`) carries the larger continuity
apparatus (Documentation Map, contract cards, gate scripts). A deliberately
complete cross-repo snapshot lives engine-side at
`_observations/OBSERVATIONS_2026_05_29_FRONTEND_INCLUSIVE_GAP_ANALYSIS.md`.
This directory is the frontend half that snapshot recommended standing up.
